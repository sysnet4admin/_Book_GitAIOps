## 체크리스트

- [ ] Strimzi operator 설치됨
- [ ] Kafka 브로커 1개 Running (KRaft 모드, Kafka 4.x)
- [ ] worker-pool 노드에 배치됨
- [ ] entity-operator memory 512Mi 설정됨
- [ ] notifications 토픽 생성됨 (3 partitions)
- [ ] Notiflex Producer: /id 요청 시 Kafka 메시지 전송
- [ ] Notiflex Consumer: 메시지 수신 로그 확인
- [ ] JOURNEY.md `현재 버전` — Kafka 버전 기재 (`kubectl get kafka -n kafka -o jsonpath='{.items[0].spec.kafka.version}'`로 조회)

## 이 단계 이후 아키텍처

```
GKE: notiflex-cluster
├── kafka namespace
│   ├── Strimzi Operator
│   ├── Kafka Broker (KRaft, worker-pool)
│   └── Topic: notifications
├── notiflex namespace
│   ├── Rollout: notiflex-api (Producer + Consumer)
│   │   └── env: KAFKA_BROKER
│   ├── Valkey
│   └── Gateway
...

데이터 흐름: /id 요청 → Valkey INCR → Kafka produce → Consumer 로그
```
