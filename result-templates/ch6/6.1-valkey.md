## 체크리스트

- [ ] Valkey standalone 모드 Running
- [ ] Notiflex가 Valkey로 ID 생성 (인메모리 → Valkey INCR)
- [ ] ID가 Pod 간 순차적으로 증가 (split-brain 해결)
- [ ] 연결 재시도 로직 동작 확인 (Pod 로그에서 확인)
- [ ] JOURNEY.md `현재 버전` — Notiflex 이미지 실제 태그 갱신 (`kubectl get rollout notiflex-api -n notiflex -o jsonpath='{.spec.template.spec.containers[0].image}'`로 조회)

## 이 단계 이후 아키텍처

```
GKE: notiflex-cluster
├── notiflex namespace
│   ├── Rollout: notiflex-api (v0.x.x)
│   │   └── env: VALKEY_ADDR, VALKEY_PASSWORD
│   ├── Valkey: standalone (데이터 공유)
│   ├── Gateway + HTTPRoute
│   └── Services
├── monitoring namespace
├── argo-rollouts namespace
└── argocd namespace

데이터 흐름: Pod → Valkey INCR → 순차 ID (전 Pod 공유)
```
