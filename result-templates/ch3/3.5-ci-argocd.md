# 3.5 CI + ArgoCD 연결 — 예상 결과

## 상태 확인

```
코드 수정 → push → CI 빌드 → 이미지 푸시 → 매니페스트 업데이트 → ArgoCD 자동 배포

$ kubectl get pods -n notiflex
(새 이미지로 Pod 교체됨)
```

## 체크리스트

- [ ] CI에서 이미지 태그 자동 업데이트
- [ ] 매니페스트 변경 → git push 자동화
- [ ] ArgoCD가 변경 감지 → 자동 배포
- [ ] 전체 파이프라인 end-to-end 동작 확인
- [ ] /update-docs 실행 안내됨

## 이 단계 이후 아키텍처

```
[코드 push] → [GitHub Actions CI] → [Artifact Registry]
                     ↓
              [매니페스트 업데이트] → [git push]
                                        ↓
                                   [ArgoCD 감지] → [GKE 자동 배포]
```
