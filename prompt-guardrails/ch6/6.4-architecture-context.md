# 6.4 claude-context/로 아키텍처 스냅샷 정리

## 사전 조건
- ch6.1~6.3 완료 (Valkey, Secret, Canary 적용)
- `JOURNEY.md`에 ch6 진행 기록이 누적되어 있음
- `docs/architecture-decisions.md`(ADR) 5장에서 도입됨

## 실행 지침

### 단계 1: claude-context/architecture.md 작성

`claude-context/architecture.md` 파일을 새로 만들어 **현재 시점**의 아키텍처 스냅샷을 정리한다. 이 문서는 AI가 매 대화에서 빠르게 전체 그림을 잡을 수 있도록 돕는 단일 페이지 요약이다.

다음 정보를 클러스터에서 직접 조회해 채운다:

```bash
# Application 목록
kubectl --context gke-sysnet4admin_book_gitaiops get app -n argocd \
  -o custom-columns=NAME:.metadata.name,SYNC:.status.sync.status,HEALTH:.status.health.status

# 네임스페이스 / 주요 워크로드
kubectl --context gke-sysnet4admin_book_gitaiops get ns
kubectl --context gke-sysnet4admin_book_gitaiops get rollout,deploy,sts -A

# Gateway / HTTPRoute
kubectl --context gke-sysnet4admin_book_gitaiops get gateway,httproute -A
```

`claude-context/architecture.md` 구성 요소:
- **현재 클러스터 토폴로지**: GKE 클러스터 이름, 리전, 노드풀 구성 (현재는 default-pool 단일)
- **컴포넌트 다이어그램** (텍스트 기반): Gateway → notiflex-api(Rollout) → Valkey, External Secrets → Secret Manager
- **트래픽 흐름**: 외부 → Gateway → Service → Rollout(Canary) → Pod
- **데이터 흐름**: API → Valkey 캐시 / API → Secret Manager(via CSI Driver)
- **GitOps 흐름**: Git push → ArgoCD auto-sync → 클러스터 상태 갱신
- **참고 문서**:
  - `docs/JOURNEY.md` — 진행 기록·도구 선택 기록
  - `docs/architecture-decisions.md` — 결정 ADR 누적
  - `CLAUDE.md` — AI 행동 규칙

### 단계 2: 3층 지식 구조 정리

`claude-context/architecture.md` 머리에 짧은 문단으로 세 문서의 역할을 명시한다:

- **CLAUDE.md** — AI에게 프로젝트 메타데이터 제공 (매 대화 자동 로드)
- **claude-context/** — AI가 참조할 현재 아키텍처 스냅샷 (자동 참조)
- **docs/architecture-decisions.md** — 결정 누적, 사람과 AI가 함께 검토

이 세 층이 분리되어야 메모리(작업 컨텍스트)·아키텍처(현재 그림)·결정(과거 누적)이 섞이지 않는다.

### 단계 3: JOURNEY.md 업데이트

`JOURNEY.md`의 진행 현황·도구 선택·리소스 섹션을 현재 상태로 갱신한다. ch6에서 도입한 Valkey, External Secrets, Argo Rollouts(Canary 전략)을 반영한다.

### 단계 4: `/update-docs` 슬래시 명령으로 커밋

`/update-docs` 스킬을 호출해 다음 변경을 한 커밋으로 묶는다:
- `claude-context/architecture.md` (신규)
- `JOURNEY.md` (갱신)

```bash
git add JOURNEY.md claude-context/
git commit -m "ch6: 아키텍처 스냅샷 정리, 캐시·시크릿·Canary 도입 반영"
git push
```

## 트러블슈팅

### `claude-context/`가 너무 커진다
→ 한 페이지(약 200~300줄) 안에 들어가도록 유지한다. 컴포넌트가 늘면 파일을 쪼개기보다 표·요약 위주로 다듬는다. 세부는 `JOURNEY.md`·ADR·매니페스트에 남기고 `claude-context/`는 항상 **현재 상태 한눈 보기**로만 쓴다.

### 정보가 ADR과 중복된다
→ ADR은 "왜 이 결정을 내렸는가"의 누적 기록, `claude-context/`는 "지금 어떻게 동작하는가"의 스냅샷이다. 결정 이유는 ADR에만, 현재 토폴로지는 `claude-context/`에만 둔다.
