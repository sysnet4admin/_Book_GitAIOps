# 2.2 Claude Code 설치 확인

## 사전 조건

- Anthropic API 키 또는 Claude Pro/Max 구독이 있어야 한다
- macOS/Linux: 터미널 사용 가능
- Windows: Git for Windows 설치 필요 (https://gitforwindows.org), Windows Terminal 권장

## 실행 지침

Claude Code를 설치하고, 정상 동작하는지 확인하고, 선택적으로 statusline을 구성한다.

### 단계 0: Claude Code 설치

독자의 OS를 확인한 후 공식 설치 명령어를 안내한다.

#### macOS / Linux

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

- 공식 설치 스크립트로 자동 업데이트를 지원한다
- 설치 후 터미널을 재시작한다
- `jq`가 없으면 macOS: `brew install jq`, Linux: `sudo apt install jq` (또는 배포판에 맞는 패키지 매니저) 안내

#### Windows

**사전 준비:**
- Git for Windows 설치 (https://gitforwindows.org) — Git Bash 포함
- Windows Terminal 사용 권장 (cmd.exe는 ANSI 색상이 깨짐)

**PowerShell에서 설치:**
```powershell
irm https://claude.ai/install.ps1 | iex
```

**CMD에서 설치:**
```cmd
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
```

- 자동 업데이트를 지원한다
- `jq` 설치: `winget install jqlang.jq` 또는 `choco install jq`

#### 대안: 패키지 매니저 (자동 업데이트 미지원)

| OS | 명령어 |
|----|--------|
| macOS (Homebrew) | `brew install --cask claude-code` |
| Windows (WinGet) | `winget install Anthropic.ClaudeCode` |

> ⚠️ 패키지 매니저 설치는 자동 업데이트가 되지 않으므로 공식 설치 스크립트를 권장한다.

#### 설치 확인

```bash
claude --version
```

정상 출력되면 설치 완료.

#### 실행 모드

실습에서는 매번 승인 없이 자동으로 진행하는 모드를 권장한다:

```bash
claude --dangerously-skip-permissions
```

- 파일 생성, 명령 실행, Git 작업 등을 승인 없이 자동 진행
- 실습용 전용 환경(GKE 클러스터, 전용 저장소)에서만 작업하므로 안전
- 일반 모드(`claude`)로 진행해도 되지만, 매 단계마다 "Allow?" 승인이 필요함을 안내

> 이 책의 모든 실습은 `claude --dangerously-skip-permissions` 실행을 기준으로 한다.

### 단계 2: Statusline 설정 (선택)

독자에게 다음과 같이 안내한다:

> 이 책의 실습에서는 **모델명, 컨텍스트 사용량, API 사용률, Kubernetes 컨텍스트, 현재 경로**를 statusline에 표시하는 설정을 제공합니다.
> 선호하는 설정이 있거나 기본 상태로 사용하고 싶다면 건너뛰어도 됩니다.
>
> statusline을 설정할까요?

#### 독자가 "예"를 선택한 경우

OS를 확인한 후 아래 절차를 진행한다.

##### 공통: 설정 파일 구성

`~/.claude/settings.json`에 statusline 설정을 추가한다:

```json
{
  "statusLine": {
    "type": "command",
    "command": "/bin/bash ~/.claude/statusline-command.sh",
    "padding": 0
  }
}
```

> ⚠️ **기존 settings.json이 있는 경우**: 기존 내용을 덮어쓰지 않는다. `statusLine` 키만 추가/병합한다. 독자에게 기존 설정이 유지됨을 안내한다.

> ⚠️ **Windows의 경우**: `command` 경로를 Windows 형식으로 변경한다:
> ```json
> "command": "bash %USERPROFILE%\\.claude\\statusline-command.sh"
> ```
> Git Bash가 설치되어 있어야 한다. Git for Windows에 포함되어 있으므로 별도 설치는 불필요한 경우가 많다.

##### 공통: statusline-command.sh 생성

`~/.claude/statusline-command.sh` 파일을 생성한다. 핵심 기능:

- **모델명 축약**: `Claude Sonnet 4.6` → `Sonnet 4.6`
- **컨텍스트 게이지**: 10칸 바 (초록 <50%, 노랑 50-70%, 빨강 ≥70%)
- **API 사용률**: `rate_limits.five_hour` 데이터에서 5시간 윈도우 사용률과 리셋 시간 표시 (색상: 초록 <50%, 노랑 50-80%, 빨강 ≥80%). Max 요금제는 `rate_limits.seven_day` 7일 사용량도 함께 표시
- **Kubernetes 컨텍스트**: 현재 컨텍스트를 축약하여 표시
- **경로 축약**: `first/…/current` 형식

출력 예시: `Sonnet 4.6 | ▓▓▓░░░░░░░ | 5h:42%(3h24m) | gke-sysnet4admin_book_gitaiops | managed-/…/notiflex-platform`

Max 요금제에서 Opus로 전환한 경우 출력 예시: `Opus 4.7 (1M context) | ▓▓▓░░░░░░░ | 5h:42%(3h24m) 7d:15% | gke-sysnet4admin_book_gitaiops | managed-/…/notiflex-platform`

스크립트 작성 시 아래 사항을 반드시 따른다:

**컨텍스트 사용량** — stdin JSON에서 직접 읽는다:
```bash
pct=$(echo "$input" | jq -r '.context_window.used_percentage // 0')
pct=${pct%.*}
```

**API 사용률** — stdin JSON의 `rate_limits`에서 읽는다:
```bash
# 5시간 윈도우 (공통)
five_hour_pct=$(echo "$input" | jq -r '.rate_limits.five_hour.used_percentage // empty')
resets_at=$(echo "$input" | jq -r '.rate_limits.five_hour.resets_at // empty')

# 7일 윈도우 (Max 요금제만 — 없으면 표시하지 않음)
seven_day_pct=$(echo "$input" | jq -r '.rate_limits.seven_day.used_percentage // empty')
```
- `resets_at`는 Unix epoch(초)이므로 현재 시각과의 차이로 남은 시간을 계산한다
- `seven_day`가 존재하면 `5h:XX%(리셋시간) 7d:XX%` 형식으로, 없으면 `5h:XX%(리셋시간)`만 표시
- 외부 API 호출 없이 stdin 데이터만 사용한다

**Kubernetes 컨텍스트** — `kubectl config current-context` 사용:
- `kubectl`이 없으면 빈 문자열 반환 (에러 미출력)
- `kubernetes-admin@cluster` → `cluster`로 축약
- AWS EKS ARN → 클러스터 이름만 추출

> ⚠️ **참고 구현**: https://github.com/sysnet4admin/managed-by-claude-code/tree/main/claude-config 의 `.claude/statusline-command.sh`를 참고한다.

##### macOS 전용 사항

- 추가 설정 없이 정상 동작한다
- `jq`가 없으면 `brew install jq`를 안내한다

##### Windows 전용 사항

> ⚠️ **Windows에서는 적극적으로 트러블슈팅한다.** 아래 예외가 자주 발생하며, 에러 발생 시 바로 해결 방안을 제시한다.

| 예외 | 원인 | 해결 |
|------|------|------|
| `bash: command not found` | Git Bash가 PATH에 없음 | `command`를 `"C:\\Program Files\\Git\\bin\\bash.exe" %USERPROFILE%\\.claude\\statusline-command.sh`로 변경 |
| `jq: command not found` | jq 미설치 | `winget install jqlang.jq` 또는 `choco install jq` 안내 |
| `kubectl: command not found` | kubectl 미설치 | K8s 컨텍스트 표시만 비활성, 나머지는 정상 동작 |
| API 사용률이 안 보임 | `rate_limits`가 stdin JSON에 포함되지 않는 환경 | 정상 — 사용률 없이 나머지만 표시되도록 스크립트가 처리함 |
| 경로에 백슬래시 | Windows 경로 구분자 | `sed 's|\\\\|/|g'`로 변환 |
| ANSI 색상이 깨짐 | Windows Terminal이 아닌 cmd.exe 사용 | Windows Terminal 사용을 권장 |

##### 선택: 모델 전환 슬래시 명령어

추가로 모델 전환 명령어를 설정할 수 있다:

| 파일 | 명령어 | 모델 |
|------|--------|------|
| `~/.claude/commands/h.md` | `/h` | Haiku 4.5 (`claude-haiku-4-5`) |
| `~/.claude/commands/s.md` | `/s` | Sonnet 4.6 (`claude-sonnet-4-6`) |
| `~/.claude/commands/o.md` | `/o` | Opus 4.7 (`claude-opus-4-7`) |

각 파일 형식:
```markdown
---
description: 설명
model: 모델ID
---

$ARGUMENTS
```

### 단계 3: 설정 확인

statusline을 설정한 경우:
1. Claude Code를 재시작하도록 안내한다
2. 재시작 후 하단에 statusline이 표시되는지 확인한다
3. 문제가 있으면 즉시 트러블슈팅한다

## 트러블슈팅

### statusline이 표시되지 않음
→ `~/.claude/settings.json`에 `statusLine` 키가 있는지 확인
→ `statusline-command.sh`에 실행 권한이 있는지 확인: `chmod +x ~/.claude/statusline-command.sh`
→ 스크립트를 수동 실행하여 에러 확인: `echo '{}' | bash ~/.claude/statusline-command.sh`

### statusline이 잘려서 보임
→ 터미널 폭이 좁으면 오른쪽 정보가 잘린다. 터미널을 넓히거나 컨텍스트 이름을 축약한다.

### Windows에서 statusline이 전혀 동작하지 않음
→ Git Bash 경로를 절대 경로로 지정한다
→ `%USERPROFILE%`이 전개되지 않으면 `C:\Users\<사용자명>\.claude\statusline-command.sh`로 직접 지정한다
→ 그래도 안 되면 statusline을 건너뛰고 진행한다 (실습 진행에 필수는 아님)
