# 9.4 GitAIOps의 출현

## 실행 지침

이 프로젝트에서 Git, AI, Ops가 어떻게 연결되어 있는지 분석한다.

`JOURNEY.md`를 데이터 소스로 활용하여 **독자의 실제 여정** 기반으로 분석한다:

- **Git**: 코드, 매니페스트, JOURNEY.md(의사결정 기록)의 저장소
- **AI**: Claude Code가 코드 생성, 문제 해결, 아키텍처 결정을 보조 → JOURNEY.md에 기록
- **Ops**: JOURNEY.md의 도구 선택에 따른 GitOps 도구가 Git의 상태를 클러스터에 동기화

세 요소가 결합되어 "살아있는 운영 표준"이 어떻게 탄생하는지 설명한다.
JOURNEY.md 자체가 Git+AI+Ops의 산물임을 보여준다.

### GitAIOps 루프 다이어그램

세 요소가 일회성 결합이 아니라 **반복 루프**로 강화되는 구조를 그림으로 보여준다:

```
독자 입력 (자연어)
        ↓
[탐색] AI가 옵션 제시 ──→ [비교] AI가 트레이드오프 정리 ──→ [실행] AI가 매니페스트·명령 생성
        ↓                        ↓                                  ↓
   JOURNEY.md 갱신         ADR 추가              Git push → ArgoCD sync → 클러스터 적용
        ↓                        ↓                                  ↓
                          가드레일·CLAUDE.md 누적
        ↓                        ↓                                  ↓
                다음 [필수입력] 진입 시 AI가 누적 자산 자동 참조
                         (루프가 한 바퀴 돌수록 학습이 빨라진다)
```

핵심은 **한 사이클이 다음 사이클을 강화**한다는 점이다. 첫 사이클은 탐색·비교에 시간이 걸리지만, 두 번째부터는 AI가 이전 결정을 알고 있으므로 곧장 실행으로 넘어간다.

### 3-프롬프트 패턴과의 연결

책 전반의 [필수입력] 패턴 — **탐색 → 비교 → 실행** 3단계 — 은 GitAIOps 루프의 한 바퀴를 표현한 것이다. 가드레일이 누적될수록 같은 패턴이 점점 짧아진다 (탐색·비교가 압축되고 실행만 남음). 회고에서 이 가속화를 짚는다.
